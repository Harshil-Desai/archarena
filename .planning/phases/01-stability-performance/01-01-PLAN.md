**1. src/lib/graph-parser.ts** -- Replace `hasGraphChanged` (line ~275-281). Remove the JSON.stringify comparison. Replace with structural comparison:
```typescript
export function hasGraphChanged(
  prev: SemanticGraph | null,
  next: SemanticGraph
): boolean {
  if (!prev) return true
  if (prev.nodes.length !== next.nodes.length) return true
  if (prev.edges.length !== next.edges.length) return true
  if (prev.annotations.length !== next.annotations.length) return true
  if (prev.nodes.some((n, i) => n.id !== next.nodes[i]?.id || n.label !== next.nodes[i]?.label)) return true
  if (prev.edges.some((e, i) => e.id !== next.edges[i]?.id || e.label !== next.edges[i]?.label)) return true
  return false
}
```

**2. src/lib/ai.ts** -- Fix line 82: change `pro: "gemini-2.5-flash"` to `pro: "gemini-2.5-pro"`. Also add `MAX_VOICE_TOKENS: 200` to the `LIMITS_AI` object (for AICOST-01 infrastructure, consumed by Phase 5).

**3. src/lib/daily-limits.ts** -- Rewrite both `checkAndIncrementDailyHints` and `checkAndIncrementDailyScores` to use atomic operations. The pattern:
- First, handle the 24-hour reset: use `prisma.user.updateMany` with `WHERE id = userId AND dailyLimitResetAt < (now - 24h)` to reset counters atomically.
- Then, attempt the atomic increment: `prisma.user.updateMany({ where: { id: userId, dailyHintsUsed: { lt: dailyLimit } }, data: { dailyHintsUsed: { increment: 1 } } })`. If `result.count === 0`, the limit was reached.
- Return remaining count by reading after increment (single extra read is fine since the write is now safe).
- Do NOT use separate read-then-write pattern. The entire check+increment must be a single updateMany call.
