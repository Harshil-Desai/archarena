# External Integrations

**Analysis Date:** 2026-04-15

## APIs & External Services

**LLM/AI Providers:**
- Anthropic Claude API - AI hints, chat replies, and scoring

  - Models: `claude-haiku-4-5-20251001` (hints/chat), `claude-sonnet-4-6` (scoring)

  - Models: `gemini-2.5-flash` (both hints and scoring)
  - Auth: `GOOGLE_GEMINI_API_KEY` (server-only)

  - Status: Currently configured but marked as backup in code comments

**OAuth Identity Providers:**

- GitHub OAuth 2.0 - User authentication via GitHub Sign-In

## Authentication & Identity

**Session Configuration:**
- Strategy: `"database"` (via PrismaAdapter)
- Adapter: `@auth/prisma-adapter` 2.11.1
- Session token stored in Prisma `Session` model
- User tier (`FREE` | `PRO` | `PREMIUM`) attached to session callback
- Implementation: `src/auth.ts`

**OAuth Flow:**
- Sign-in page: `/login` (custom route)
- Callback handling: `/api/auth/[...nextauth]` (auto-generated by NextAuth)
- Session refresh: Token-based; tier read fresh from DB on each API call per CLAUDE.md

## Environment Configuration

**Required env vars (server-only):**
- `ANTHROPIC_API_KEY` - Anthropic API key
- `DATABASE_URL` - PostgreSQL connection string
- `NEXTAUTH_URL` - Base URL for NextAuth (e.g., http://localhost:3000)
- `NEXTAUTH_SECRET` - Random secret for session encryption
- `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` - Google OAuth credentials
- `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` - GitHub OAuth credentials
- `LEMONSQUEEZY_API_KEY` - LemonSqueezy API key for checkout/webhooks
- `LEMONSQUEEZY_STORE_ID` - LemonSqueezy store ID for checkout
- `LEMONSQUEEZY_WEBHOOK_SECRET` - HMAC secret for webhook signature verification

- `NEXT_PUBLIC_LEMONSQUEEZY_PRO_VARIANT_ID` - Product variant ID for Pro tier
- `NEXT_PUBLIC_LEMONSQUEEZY_PREMIUM_VARIANT_ID` - Product variant ID for Premium tier

**Secrets location:**
- `.env.local` in development

## Webhooks & Callbacks

**Incoming (Webhook Handlers):**
- LemonSqueezy subscription events → `/api/billing/webhook`
  - Event signature verification: HMAC-SHA256
  - Payload events: subscription created, updated, expired, cancelled
  - Updates user tier, subscription status, and renewal dates in PostgreSQL

**Outgoing (Integration Callbacks):**
- LemonSqueezy checkout redirect URL: `{NEXTAUTH_URL}/billing/success`
- OAuth redirect URIs: Google/GitHub configured in provider dashboards

**API Endpoints:**
- `POST /api/ai/hint` - Request AI hint (Anthropic or Gemini)
- `POST /api/ai/score` - Stream scoring result
- `POST /api/session/[id]/chat` - Chat with interviewer
- `POST /api/billing/checkout` - Create LemonSqueezy checkout session
- `POST /api/billing/webhook` - LemonSqueezy webhook receiver
- `GET /api/session/[id]` - Fetch session by ID

## API Request/Response Patterns

**AI API Calls:**
- Input: SemanticGraph (from `src/lib/graph-parser.ts`, never raw tldraw JSON)
- Truncation before sending: `truncateGraphForAI()` in `src/lib/ai.ts`
- Rate limiting: Per-user daily limits stored in Prisma User model
- Error handling: Quota errors (429) trigger specific response; free tier limit (403) returns `free_limit_reached`

**Scoring Streams:**
- Response type: `ReadableStream` (Server-Sent Events)
- Pre-flight error handling: Quota errors caught before stream creation
- JSON parsing: Sanitized response (strip markdown fences, validate JSON end)

**LemonSqueezy Integration:**
- Variant ID mapping: PRO and PREMIUM variant IDs mapped to tier enum
- Webhook signature: SHA256 HMAC with timing-safe comparison
- Custom data: User ID passed via `custom.user_id` field in checkout

## Rate Limiting

**Hints:**
- Per-session: FREE tier = 5 hints, PRO/PREMIUM = unlimited
- Daily: FREE = 10/day, PRO = 100/day, PREMIUM = 100/day
- Enforcement: Database queries in `src/lib/daily-limits.ts`

**Scores:**
- Per-session: FREE = 1, PRO = unlimited, PREMIUM = unlimited
- Daily: FREE = 3/day, PRO = 20/day, PREMIUM = 20/day
- Enforcement: Database queries in `src/lib/daily-limits.ts`

**Prompts:**
- Available prompts: FREE = 5, PRO = unlimited (from `src/lib/limits.ts`)
